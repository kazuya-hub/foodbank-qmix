
# 分析用コードについて

このディレクトリには、東京高専の榊原和哉が2023年度と2025年度の研究で実験結果の分析に使用したコードが置いてあります

## `for_sacred_logs.py`

2023年度論文の実験で使用したもので、sacredで記録された状態・行動・価値関数の出力などを観察するために使用したコードです。sacredのログ自体が機械的な処理にあまり向いていないのと、複数runを同時に回した時にログが混ざる問題もあって、研究の後の方ではsacredログはあまり使いませんでした。このコードは分析結果の再現のために残してありますが、再利用は推奨しません。

### 使い方

- ローカルのsacredログを読み込むので、過去の実験結果を分析したい場合は `実験ログデータ引き継ぎ/sacred` からログをコピーしてきて `results/sacred` に配置する必要があります。詳しくは `実験ログデータ引き継ぎ/ログデータの引き継ぎについて.md` を参照してください。
- runの指定は、対応するrunのディレクトリをコード内にハードコーディングしています
- 読み込みに成功すると、2つのウィンドウが開きます。2つのウィンドウは連動していて、キーボードの矢印キーでエピソードやタイムステップを移動できます。上下矢印でエピソードを移動、左右矢印でエピソード中のステップを移動できます。
    - 1つ目のウィンドウは状態（観測）と行動を示すもので、一番大きい緑色のヒートマップは各施設が持っている食品を示します（縦軸が施設、横軸が食品）。緑色のヒートマップ内で枠がオレンジ色になっている部分は、そのステップで該当する施設が該当する食品を取得したことを意味し、赤色は食品を選択したが実際には取得できなかった（他のエージェントが先に取ってしまった）ことを意味します。左の茶色いヒートマップは、各行に対応する施設が食品を取得した (茶色) / 選択したが取得できなかった (オレンジ) / 待機した (白) ことを意味します。上の茶色いヒートマップは、各列に対応する食品のフードバンクの在庫を示します。
    - 2つ目のウィンドウは各エージェントの価値関数 (`Q_a`) の出力を示すもので、縦軸はエージェント（施設）、横軸は行動です。

## `runtime_and_success_rate.py`

2023年度論文の実験で使用したもので、wandbのログを分析して、施設数・食品種数の変化に伴う計算時間や学習の成功率を観察するために使用したコードです。wandbモジュールの機能を使ってクラウド上に保存されているログを取得・分析します。プロジェクトの設定もpublicにしているので特別な手順を踏まなくても `runtime_and_success_rate.py` を実行するだけで結果を再現できるかと思いますが、もしエラーが出たりしたら遠慮なく相談してください。

### 使い方

- `runtime_and_success_rate.py` を実行するだけです。
- runの絞り込み条件やデータの加工方法についてはコード内のコメントを参考にしてください
- 計算時間は、全く同じ設定でも実行するマシンごとに異なると予想されるため、 `hostname` で識別して別々に集計しています
- 学習結果自体はどのマシンでも同じになるはずなので、学習成功率の集計ではマシンを区別しません

## `MixingNetwork_weights_and_biases.ipynb`

2025年度論文の4.2に示した図を作成するために使用したもので、wandbのログを分析して、 Mixing Network の重み・バイアスの値の分布をグラフに起こします。

### 使い方

- 高速化のため、クラウド上ではなくローカルのログから取得している部分があるため、過去の実験結果を分析したい場合は `実験ログデータ引き継ぎ/wandb` からログをコピーしてきて `wandb` に配置する必要があります。詳しくは `実験ログデータ引き継ぎ/ログデータの引き継ぎについて.md` を参照してください。
- 実行自体は、 `MixingNetwork_weights_and_biases.ipynb` 中のセルを上から順番に実行するだけです。ファイル内のコメントなども参考にしてください。

## `analysis_with_array_logger.ipynb` / `analysis_with_array_logger.py`

2025年度論文で、array_loggerによって記録された各種Tensor (ndarray) を読み込み・分析する際に使用したものです。 ipynb の方でTensor用のビューワーを作りましたがうまく動かず、pyの方も使いました。

### 使い方

- array_loggerがローカルに保存したログを読み込むので、過去の実験結果を分析したい場合は `実験ログデータ引き継ぎ/array_logs` からログをコピーしてきて `results/array_logs` に配置する必要があります。詳しくは `実験ログデータ引き継ぎ/ログデータの引き継ぎについて.md` を参照してください。
- ipynbの方は上の方のセルから順に実行してください。
- ファイル内にも処理の内容のコメントや、一度使った後コメントアウトした処理などがあるので参考にしてください。
